# Alpha 冲刺计划

## 核心目标

14 个工作日内完成「前后端一体化可运行 Alpha 版本」，实现核心功能闭环、稳定性验证与基础体验优化

## 阶段划分与关键任务

### 第 1-2 天：需求对齐与基础准备

- 需求梳理：最终确认核心功能清单（快速归集、智能解析、智能聚合、智能共创、成长感知与激励）
- 技术对齐：
  - 前后端接口规范冻结（基于 Swagger 文档，更新嵌入式环境下的接口地址、请求 / 响应格式）
  - 后端使用Springboot
- 环境准备：前端搭建 Electron+Vue3 开发环境，后端完成Springboot框架搭建与本地测试

### 第 3-6 天：功能开发攻坚（核心开发）

- 后端（4 天）：
  - 第 3 天：完善收集，标签归类逻辑
  - 第 4 天：核心功能开发
  - 第 5-6 天：AI 任务处理优化（对接简易 AI 生成接口，完善异步任务状态跟踪）
- 前端（4 天）：
  - 第 3-4 天：基础页面开发（登录 / 注册页、首页、笔记列表页，实现路由守卫与登录态管理）
  - 第 5-6 天：核心功能页面开发（笔记编辑器、AI 生成入口页，含参数配置、加载状态、结果展示组件）
- 同步动作：每日站会同步进度，解决跨端依赖问题（如接口字段变更、前端组件设计争议）

### 第 7-9 天：接口联调与功能闭环

- 第 7 天：基础接口联调（收藏列表查询 / 创建，确保数据互通）
- 第 8 天：核心功能联调（编辑 / 删除、AI 生成任务提交 / 结果回调）
- 第 9 天：功能闭环验证（全流程走通：安装→收藏内容→创建想法→AI 生成→查看 / 编辑笔记→退出），修复联调中的接口报错、数据格式不匹配等问题

### 第 10-11 天：一体化打包与环境适配

- 第 10 天：
  - 前端：Electron 整合后端 JAR，实现进程联动（启动 Electron 自动拉起后端，关闭 Electron 终止后端进程）
  - 打包配置：完成 Electron-builder 配置（含 JRE 打包、资源路径配置、exe 图标 / 安装流程设计）
- 第 11 天：
  - 多环境测试：在 Windows 本地验证 exe 安装包的可用性（安装、启动、功能运行）
  - 路径 / 进程问题修复（如打包后 JAR 路径错误、后端进程未正常终止等）

### 第 12-13 天：Bug 修复与体验优化（打磨优化）

### 第 14 天：Alpha 版本验收与交付

- 产出交付物：Alpha 版本安装包（Windows exe）、版本说明文档
- 源码归档：整理前后端源码，提交代码仓库

## 交付物

1. 可直接安装的 Alpha 版安装包（Windows exe ，含前后端 + 嵌入式依赖，免额外安装 Java/MySQL/Redis）
2. 版本说明文档（安装步骤、核心功能清单、已知限制、问题反馈渠道）
3. 完整源码包（含前后端代码、打包配置文件、数据库初始化脚本）

# 前期工作总结与思考及改进方案

## （一）前期工作总结

- 调整项目需求
- 完成项目核心需求调研与定位，明确 “快速归集、智能解析、智能聚合、智能共创、成长感知与激励” 五大核心功能方向
- 确定了适配桌面端场景的技术栈（前端 Electron+Vue3、后端 SpringBoot），初步搭建了技术架构雏形，完成了基础接口规范的初稿设计；
- 完成了团队角色分工，明确了各岗位核心职责，为冲刺阶段的协作模式铺垫了基础。

- 接口联调准备不足：虽有初步接口规范，但未提前进行模拟测试，前后端对字段格式、请求 / 响应逻辑的理解存在潜在偏差；
- AI 对接缺乏预案：对 AI 生成接口的异步任务状态跟踪、异常处理（如超时、失败重试）考虑不全面，未制定适配方案；
- 测试同步滞后：测试用例未随需求同步设计，前期未介入技术方案评审，可能导致测试覆盖不全面、功能验证滞后。

## （二）改进思路与方案

| 改进方向        | 具体方案                                                     |
| --------------- | ------------------------------------------------------------ |
| 前置接口验证    | 基于 Swagger 文档冻结接口规范后，前端用 Mock.js 搭建模拟接口环境，后端启动本地测试服务；完成接口字段、请求方式的交叉校验 |
| AI 接口适配优化 | 后端封装统一 AI 调用工具类，包含超时重试、异常捕获、状态回调；前端设计加载状态、失败重试按钮、结果异常提示 UI |
| 测试同步介入    | 测试人员从依据《Alpha 版本功能清单》编写测试用例；联调阶段同步执行测试，跟踪修复进度 |
